# tool-specific-tips — create-gadget

Nothing fancy. Gadget pages are pure Markdown + frontmatter. No APIs to hit, no scripts to run.

- **Image (optional):** save product photos to `static/images/gadgets/<slug>.jpg` and reference via a standard Markdown image `![](/images/gadgets/<slug>.jpg)` in the body. Featured images are not used for gadgets by default.
- **Specs research:** if the user asks you to fill specs and you have web access, the manufacturer's product page or Wikipedia entry is the cleanest source. Quote figures verbatim; don't estimate.
- **Do not invent tips.** Leave "Tips and tricks" empty and prompt the user to add their own over time. The gadget section is a personal notebook, not a product review site.
